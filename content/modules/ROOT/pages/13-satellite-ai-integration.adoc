== Satellite AI Integration

Red Hat Satellite can integrate with AI-powered tools to enhance
infrastructure management. In this module, we'll explore how Satellite
AI integration can help streamline operations, provide intelligent
recommendations, and improve decision-making for system administrators.

=== Benefits of Satellite AI Integration

Satellite integrates with Large Language Models (LLMs) to provide intelligent insights and recommendations for system management. This integration is enabled through the Model Context Protocol (MCP). Red Hat Satellite is shipped with an MCP server that can be configured as an interface for LLMs, providing access to the Satellite API. This allows LLMs to interact with Satellite's data and functionality, enabling them to generate context-aware responses and recommendations based on the current state of the infrastructure.

The integration of the Model Context Protocol (MCP) and AI in Red Hat Satellite provides significant value by augmenting human expertise with AI-driven automation.

For System Administrators: It enables conversational management by allowing admins to use natural language queries via a Large Language Model (LLM) to interface securely with Satellite's API. Admins can ask the AI to browse the Satellite inventory, categorize reports, and trigger remote execution jobs for patching.

For IT Decision Makers: It maximizes team productivity, especially in tight labor markets, by helping to bridge the technical skills gap. AI-assisted workflows allow junior staff to handle routine maintenance and patching, organizations can free up their senior talent to focus on high-value architecture and innovation projects

=====
`opencode`
[%collapsible]
====

The `rhel-ai.lab` system is pre-installed with `opencode` client. `opencode` is a tui based command line tool that allows users to interact with the MCP server and LLMs.

This lab is configured to use the `minimax-m2` model, which is a powerful LLM that can provide intelligent insights and recommendations based on the data it receives from the Satellite MCP server.

First, click on the `rhel-ai.lab terminal` tab.

.rhel-ai.lab terminal
image::rhelaitab.png[]

Click on the run button to run the following command to start `opencode`:
[source,bash,run]
----
opencode
----

[NOTE]
--
Opencode has a bug which emits spurious error messages upon launch. The messages can be removed with the following procedure.

.opencode bug
image::opencodebug.png[]

. Press ctrl+p.

. Press the escape key.

This will open the opencode options menu. When you exit the menu, the error messages will be cleared.
--

You may choose to use the web gui for Opencode instead of the terminal interface. Click on the tab `+rhel-ai.lab Web GUI+` to access the web interface. Click on the project called +/+ to open the prompt.

.Opencode WebGUI
image::opencode_webgui.png[]

Ask a question about the Satellite inventory
[%collapsible]
====
In the `opencode` prompt, ask the following question about the Satellite inventory:

[source,js]
----
What is the total number of hosts in the Satellite inventory?
----

The output will look similar to the image below.

.Satellite inventory question and answer
image::totalnumberhostssatelliteinventory.png[]

In the output, we can see that the LLM was able to query the Satellite MCP server and retrieve the total number of hosts in the Satellite inventory, which is 2 in this case.

=====
Trigger a remote execution job for patching
[%collapsible]
====
In the `opencode` prompt, tell the LLM to update rhel1:
[source,js]
----
Update all software on rhel1.
----

.Host patching remote execution job
image::appliederrata.png[]

It may take a couple minutes before `+rhel1+` is fully updated.

IMPORTANT: The efficacy of the LLM's ability to apply commands to run operations like patching depends on the quality of the LLM. In this lab, we are using the `minimax-m2` model because it's cheap. It's not particularly good at interpreting intent and thus fails at many tasks. For better results, use a higher quality model.

You can try asking the LLM to perform other tasks.

For example:

[source,js]
----
Show me the number of entitlements I have and used
----

[source,js]
----
What kernel versions are running on my hosts?
----

When you're done experimenting, you can conclude this lab in the next section.
