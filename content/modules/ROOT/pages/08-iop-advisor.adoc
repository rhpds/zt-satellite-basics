== Red Hat Lightspeed for Satellite

Red Hat Lightspeed services for disconnected/air gapped Satellite environments are now available and includes the following services:

. Advisor
. Vulnerability

Red Hat Lightspeed advisor is a service that analyzes your systems and recommends actions to improve your security and performance. Red Hat Lightspeed vulnerability service is a service that tracks vulnerabilities and recommends actions to improve your security.

The Satellite server in this lab is disconnected from the internet. You'll review the advisor service in this lab. There is an optional section at the end of this lab to review the vulnerability service. The vulnerability service is in technology preview in Satellite 6.19.

=====
.Update the cvemap
[%collapsible]
====
Before we delve into the vulnerability service, we need to update the CVE map file.

The CVE map is a file that contains the mapping of CVEs to packages. This file is used by the vulnerability service to determine which CVEs affect which packages. The CVE map is updated regularly to ensure that the vulnerability service has the most up-to-date information about CVEs.

To update the CVE map, run the following command in the `+satellite.lab terminal+`.
[source,bash,run]
----
curl -o cvemap.xml https://security.access.redhat.com/data/meta/v1/cvemap.xml
----

This command will download the latest CVE mapping file from Red Hat. The cvemap.xml file maps every CVE to Red Hat product vulnerability data.

Each CVE entry contains:
  - CVE ID and description
  - Affected releases with CPE identifiers
  - Product names, release dates, and advisories (RHSA-xxxx)
  - Package names and fixed-in versions
  - Vulnerability status (e.g., "Not vulnerable" with explanation)

To import the CVE map into Satellite, run the following command:
[source,bash,run]
----
cp cvemap.xml /var/lib/foreman/
----

.Vulnerability service
[%collapsible]
====

The Vulnerability Service powered by Red Hat Lightspeed is the primary method for tracking security issues involving viewing and triaging CVEs (Common Vulnerabilities and Exposures).

The system displays detailed information about CVEs, including the CVE ID, description, publish date, severity, and CVSS score.

We want to provide a security-first experience for CVEs in Satellite, to complement the current errata-first view of CVEs.

You can access the vulnerability service by clicking on the `+Vulnerability+` menu.

.Vulnerability menu
image::vulnerability-menu.png[../assets/vulnerability-menu]

When you click on a CVE, you will be taken to the CVE details page. Here you can see the details of the CVE, including the description, publish date, severity, and CVSS score.

.CVE details
image::cve-details.png[../assets/cve-details]

Later in this lab, you'll remediate this CVE through the errata update process.

=====
.Disable a vulnerability
[%collapsible]
====

It might be the case that a particular CVE is not relevant to your environment. You can disable the CVE to reduce the "noise" of spurious CVEs.

Click on a CVE to view the details.

.CVE details
image::cve-details2.png[../assets/cve-details]

Navigate to the `+burger+` stack button and click on it.

.Burger button
image::burger-button.png[../assets/burger-button]

To disable a CVE, click on the `+Disable Vulnerability analysis+` button on the CVE details page.

.Disable CVE
image::disable-cve.png[../assets/disable-cve]

From now on, the disabled CVE will not but shown.
